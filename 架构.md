## 部署架构

```
                 HTTPS
┌───────┐ ──────────────────→ ┌──────────────────────────────────┐
│ 浏览器 │                     │         fnOS (trim_http_cgi)      │
│       │ ←── HTML/JSON ──── │                                    │
└───────┘                     │  / → index.cgi → serveUIHTML()    │
                              │  /api/ → index.cgi → ProxyToDaemon│
                              │              ↓                    │
                              │    Unix socket (daemon.sock)       │
                              │              ↓                    │
                              │  ┌──────────────────────────────┐ │
                              │  │  守护进程 (root, 唯一DB写入)   │ │
                              │  │  ├─ HTTP API Server          │ │
                              │  │  ├─ 带宽采集 (1s)            │ │
                              │  │  ├─ 连接追踪                 │ │
                              │  │  └─ WireGuard 控制 (wgctrl)   │ │
                              │  └──────┬───────────────────────┘ │
                              │         ↓                         │
                              │  ┌──────┴──────┐                  │
                              │  │  wg-server.db │ (SQLite WAL)   │
                              │  └──────────────┘                 │
                              └────────────────────────────────────┘
```

**关键设计：** CGI 零 DB 操作，所有请求通过 Unix socket 转发给守护进程。守护进程是唯一 DB 写入者。

## 数据库

| 表 | 用途 | 关键列 |
|---|------|--------|
| `users` | 用户密钥和配置 | `private_key`, `public_key`, `preshared_key`, `internal_ip` |
| `bandwidth_history` | 带宽采样 | `user_id`, `ts`(ms), `rx_speed`, `tx_speed` |
| `connection_log` | 上下线记录 | `user_id`, `connected_at`(ms), `disconnected_at`(ms) |
| `config` | 键值配置 | `key`, `value` |
| `system_log` | 运行日志 | `level`, `message`, `created_at`(ms) |

**特点：**
- 所有时间戳统一为 `INTEGER` 毫秒（UnixMilli）
- WAL 模式 + busy_timeout 5000ms 防锁
- `Init()` 启动时自动迁移旧 TEXT 时间戳 → INTEGER

## 全局写缓冲区

`db/buffer.go` — 所有定期写入统一管理，消除各模块单独写 DB。

```
每秒采集:
  BufferedSaveBandwidthPoint()  → pointBuf (内存)
  BufferedSessionTraffic()      → sessionTraffic (内存)
  Log()                         → writeBuf (内存)

每 10 秒 FlushBuffer() — 单事务:
  ├─ batch INSERT bandwidth_history (预编译语句)
  ├─ UPDATE connection_log (会话流量)
  └─ 执行 writeBuf (日志等)

一次性任务 (SetConfig / CreateUser / DeleteUser / RecordConnection):
  → FlushBuffer() 先刷缓冲区
  → 直接写 DB
```

## 图表数据流

```
前端                             后端
────────────                      ────────
首次加载:
  since = now - 1h                → getBandwidthHistoryAgg()
  (无 since 参数)                    SELECT ... WHERE ts > 1h_ago
                                   → 100 采样点

每秒增量:
  since = chartBuf[last].ts         → CountBandwidthAfter(since)  # DB + buffer
                                     maxPoints = min(100, count)
                                     SELECT ... WHERE ts > since
                                   → 增量新点 (含 buffer 中未 flush 的)

前端去重 (Set p.ts) → 追加到滚动缓冲区 (最多 200 点)
```

**比较符对齐：** `CountBandwidthAfter` 用 `ts >`（严格大于），查询也用 `ts >`，两端一致。

## 连接追踪

守护进程每次采集遍历 WireGuard 对等端：

```
prevHS := lastHandshake[userID]       # 上次握手时间 (秒)

if prevHS == 0 && latestHandshake > 0:
    HasActiveConnection?              # 检查 DB 中是否有活跃连接
    No  → RecordConnection()          # 一键写入（先 FlushBuffer）

elif prevHS > 0 && latestHandshake == 0:
    UpdateConnectionOnDisconnect()    # 记录离线时间+流量

lastHandshake[userID] = latestHandshake

# 会话流量
BufferedSessionTraffic(userID, peerRx, peerTx)  # 进 buffer，10s flush
```

## 前端技术栈

- React 18 + TypeScript
- Recharts（折线图）
- Axios（API 请求，baseURL 动态检测 CGI 路径）
- Vite + vite-plugin-singlefile（单 HTML 内联）

**关键实现：**
- `api.ts` 动态检测 `window.location.pathname` 中 CGI 前缀
- 图表增量：`chartBuf.current`（ref 滚动缓冲区）+ `chartVer`（state 强制重渲染）
- 非活跃 tab 停止拉取（`useEffect` 返回 `clearInterval`）

## 构建与部署

| 步骤 | 命令 |
|------|------|
| 编译后端 | `cd app/backend && GOOS=linux go build -ldflags="-s -w"` |
| 编译前端 | `cd app/ui-src && npm run build` |
| 打包 fpk | `bash build.sh`（自动版本递增） |
| 部署 | 上传二进制到 `/vol1/@appcenter/wg-server/backend/`，HTML 到 `ui/`，重启 trim_http_cgi |
| 守护进程 | systemd `wg-server-daemon.service` 管理，或直接 `wg-server daemon` |

## 关键技术决策

| 决策 | 原因 |
|------|------|
| SQLite (modernc.org/sqlite) | 纯 Go，无 CGO，可交叉编译 |
| wgctrl 库 | 不依赖系统 `wg` 命令 |
| Unix socket 代理 | 消除多进程 SQLITE_BUSY |
| 全局写缓冲区 | 所有定期写入一个事务，减少磁盘 IO |
| 毫秒时间戳 | 无时区歧义，整数高效比较 |
| 单 HTML 内联 | 解决 CGI 子路径路由问题 |
| systemd 守护进程 | 崩溃重启、开机自启 |
