开发一个 WireGuard 服务端系统，用于异地组网。
运行在飞牛 fnOS 上，打包成 fpk，有图形界面。

---

## 用户管理

- 用户增删改查
- 删除用户后自动重新应用 WireGuard 配置（移除对等端，强制下线）
- 创建用户时自动生成密钥对（私钥 + 公钥）和预共享密钥（PresharedKey）
- 添加用户时自动提示最小的未被使用 IP（从配置的网段自动分配）
- 每个用户独立密钥对，互不干扰
- 用户列表显示：用户名、内部IP、公钥、在线状态、实时上下行速度（B/s）
- 用户详情显示：全部信息（含私钥、预共享密钥、公钥）、累计总流量、实时速度、在线时长、外网IP:端口
- 用户详情中的一键导出客户端配置（.conf 文件下载），配置中包含 PresharedKey
- 导出弹窗显示：服务端公钥、服务端地址、客户端IP、预共享密钥、DNS、完整配置文件

## 实时监控

- 仪表盘总带宽（上下行）实时图表，默认 1 秒刷新，曲线图展示
- 当前在线用户列表
- 内外网 IP、在线时间、上下行速度、本次流量
- 用户详情中包含一段时间内的带宽（上下行）曲线图
- 带宽历史记录每秒采样一次，默认保存 7 天（可配置天数）

## 历史记录

- 用户连接记录：用户、内外网IP、上下线时间、流量使用

## 服务控制

- 服务启动/停止/重启
- IP 网段设置（默认 192.168.5.1/24，用户从 192.168.5.10 开始分配）
- 端口设置（默认 51820）
- WireGuard 其他配置：接口名称、私钥/公钥、DNS、MTU、PostUp/PostDown
- 服务端域名/公网 IP 配置（客户端导出 Endpoint 使用此值）
- 所有配置备份导出和恢复（JSON 格式）
- 开机自启（可配置，底层通过 systemd 服务实现）

## 系统信息

- 当前版本号显示
- 程序 CPU 占用、内存占用
- 服务运行时间
- 系统总内存、可用内存、使用率
- WireGuard 内核模块是否加载
- 内核版本显示
- **WireGuard 接口状态**（wgRunning：接口是否 UP）
- **监控守护进程状态**（monitorRunning：后台进程是否运行中）
- 系统信息可配置刷新间隔（1秒～不刷新选项）

## 日志管理

- 程序运行日志查看（分页）
- 日志按级别过滤（INFO/WARN/ERROR）
- 日志关键词搜索
- 日志导出（当前筛选结果导出为文本文件）
- 日志空间占用显示
- 自动滚动到最新日志
- 关键词高亮
- 自动清理（保留最新 10000 条）

## UI 要求

- 使用 iframe 类型窗口显示在飞牛应用中心
- 不同类型功能分在不同 Tab 中（仪表盘、用户管理、配置、系统）
- IP 和网段默认可配置，添加用户时自动 hint
- 仪表盘和用户详情默认 1 秒自动刷新
- 图表使用折线图（LineChart），不用面积图或柱状图

## 技术架构

### 后端

- 语言：Go
- 数据库：SQLite（使用 modernc.org/sqlite，纯 Go，无需 CGO）
- WireGuard 控制：wgctrl-go 库（不依赖系统 wg 命令）
- CGI 运行模式（无独立端口，通过 trim_http_cgi 调用）
- 时区统一为 Asia/Shanghai
- 所有时间戳显示 UTC+8

### 架构演进

第一阶段（直接 CGI 操作 DB）：
- CGI 进程直接初始化 SQLite、持有 HTTP 路由、读写数据库
- 守护进程只做带宽采集 + WireGuard 操作
- 问题：CGI + 守护进程两个进程并发访问 SQLite → SQLITE_BUSY

第二阶段（CGI → Daemon Unix socket 代理）：
- CGI 零 DB 操作，只做 HTTP 封装
- 所有 API 请求通过 Unix socket（daemon.sock）转发给守护进程
- 守护进程是唯一 DB 写入者，彻底消除 SQLITE_BUSY
- 守护进程启动本地 HTTP API 服务，挂载完整路由
- CGI 连接失败时自动启动守护进程（Setpgid 脱离 CGI 进程组）

### 守护进程（Daemon）

- 后台常驻进程，以 root 权限运行
- 通过 systemd 服务（wg-server-daemon.service）管理：
  - Restart=on-failure，崩溃后 3 秒自动重启
  - systemctl enable 开机自启
- 职责：
  - 处理所有 DB 读写操作（通过本地 HTTP API）
  - 每秒采集带宽数据并写入 DB
  - 通过 wgctrl 读取对等端信息，写入 peers.cache 供 CGI 读取
  - 处理配置触发文件（config.trigger）：init/apply/stop
  - 自动生成服务端密钥对（首次启动时检测为空则生成）
- 生命周期脚本 pkg/cmd/main 管理，降级兼容直接 nohup 启动

### CGI 代理

- 从 PATH_INFO 提取 API 路径
- 根路径 `/` 直接返回前端 HTML（serveUIHTML）
- API 路径 `/api/...` 转发给守护进程的 Unix socket
- 连接失败时自动启动守护进程（最多 5 次重试，间隔 200ms）
- 守护进程未运行时先启动再重试

### 进程通信

- CGI → Daemon：Unix socket（daemon.sock），HTTP 协议
- Daemon → CG I：peers.cache 文件（JSON，每秒更新）
- CGI → Daemon（WireGuard 操作）：config.trigger 文件触发

### 前端

- 框架：React + TypeScript
- 构建：vite + vite-plugin-singlefile（所有 JS/CSS 内联到单 HTML）
- API 请求：baseURL 动态检测页面路径
- 缓存控制：Cache-Control: no-cache

### 安装/部署

- fpk 打包：build.sh（含版本自动递增：--patch/--minor/--major/指定版本）
- install_callback：
  - 设置可执行权限
  - 创建数据目录
  - 安装 wireguard-tools（降级依赖）
  - 启用 IP 转发（sysctl）
  - **注册 wg-server-daemon.service 到 systemd**
  - 重启 trim_http_cgi
- 安装图标：make_icons.sh
- 出口网卡自动检测（读取 /proc/net/route）

### 测试

- 测试脚本：test_api.py（18 个测试用例）
- 通过 SSH 在目标机器上执行
- 测试完成后清理测试数据和恢复系统配置
- 可生成一些模拟使用记录用于查询
